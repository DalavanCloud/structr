Introduction

structr is a free, open-source content management system written in Java using Neo4j, an embeddable NOSQL graph database.

By design, structr is modular, distributed and easy to use.

Quick Starter Guide

Running structr is fairly simple. In the following example, the installation on a standard UNIX host is described. To adapt this to other operating systems, you may have to change the commands in the startup and shutdown scripts.

  1. Make sure you have a Java 1.6 runtime environment (JRE) in place. You can find out by typing

    java -version

    on the command line. This should display something like

    user@host:~$ java -version
    java version "1.6.0_20"
    OpenJDK Runtime Environment (IcedTea6 1.9.7) (6b20-1.9.7-0ubuntu1)
    OpenJDK 64-Bit Server VM (build 19.0-b09, mixed mode)
    user@host:~$

  2. Download the current version (one ZIP file) and save it anywhere on your disk.
  3. Extract the downloaded ZIP file. This will create a directory named 'structr'.

    user@host:~$ unzip structr.zip
    Archive:  structr.zip
       creating: structr/
      inflating: structr/structr.conf
       creating: structr/modules/
      inflating: structr/modules/modules.conf
      inflating: structr/modules/structr-core-0.2.jar
      inflating: structr/modules/structr-module-web-0.2.jar
      inflating: structr/modules/structr-core-ui-0.2.jar
      inflating: structr/startup
       creating: structr/files/
      inflating: structr/shutdown
       creating: structr/db/
       creating: structr/lib/
      inflating: structr/lib/structr-webapp-0.2-standalone.jar
    user@host:~$

  4. Enter the 'structr' directory and run the 'startup' script.

    user@host:~$ cd structr
    user@host:~/structr$ ./startup
    user@host:~/structr$

    This will create two files:
    • structr.log: Application log file
    • structr.pid: Containing the process id of the java process
  5. Point your browser to http://localhost:8888/structr/"
  6. Login as 'superadmin' with the password 'sehrgeheim'.
  7. Follow the instructions on the setup page.
  8. To stop, just run the 'shutdown' script. Instead of using the shutdown command, you may just kill the respective java process.

FAQ

Q: Why another CMS?

A: Because there was no CMS with all the following attributes:

  • Easy to install
  • Java-based
  • Embedded graph database
  • Embedded application server
  • Modular, easy to extend (no need to change database schema for new add-on modules)
  • Distributed storage ("cloud-ready")

Q: What does "modular" mean for structr?

"Modular" means that structr is made of several software components, which are independent from each other. The following modules are part of a minimum installation:

  • structr-core: Core classes for entities, services, commands, and some helper classes
  • structr-core-ui: Classes, templates, CSS, JS and images for the basic admin user interface
Some basic CMS functionality is contained in an add-on module:
  • structr-module-web: Entity and UI classes, UI components for basic content management functionality

Q: Can I run structr without config file?

Yes. Just put the configuration parameters into the web.xml, see the following example:

<context-param>
    <description>Local filesystem path for embedded Neo4j database files
    <param-name>database.path</param-name>
    <param-value>/opt/structr/db</param-value>
</context-param>
